基本概念

entry: 依赖的入口文件（多入口，单入口），从入口文件出发，遍历整棵依赖树，将所有的依赖遍历完成后形成一个打包后的文件。
output: 指定打包的输出文件（名称，在哪里）（filename, path), 通过占位符确定多入口配置，打包文件的名称的唯一。

webpack基础打包

如何解析ES6, React

使用babel-loader, 在.bablerc中使用@babel/preset-env @babel/preset-react

如何解析css,less,scss

使用style-loader, css-loader,然后按照解析less还是scss添加less-loader或者scss-loader

如何解析图片，字体

使用file-loader 或者url-loader(在文件较小的情况下，会转成base64)

webpack文件监听

  • –watch命令 缺点 需要手动刷新浏览器。监听原理：轮询去判断每个文件的最后修改时间是否发生变化。当然存在文件发生变化时，也不会立刻触发更新，而是先缓存起来，等到一定时间在统一更新。
  • webpack热更新 (webpack-dev-server+HotModuleReplacementPlugin) 打包文件放在内存里面.
    PS：配置了 hot: true 会自动引入webpack.HotModuleReplacementPlugin

    启动阶段：你的代码由webpack Compiler 编译打包后，给bundle server 启动一个本地服务器，共浏览器访问。1-2-A-B
    修改阶段：你的代码修改后，经过webpack Compiler打包后，传给HMR server,然后通过HMR server传给HMR Runtime 最后由HMR runtime注入（websocket)到浏览器，更新文件变化。1-2-3-4
